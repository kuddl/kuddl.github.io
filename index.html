Vorwort

Es ist zwar nicht das erste Mal, dass dokumentiert wird, wie ein Blog entsteht, aber für mich ist es ein Anfang. Ich sehe diese Seiten nicht als "lesenswerte Artikel", sondern eher als ausführliche Commitmessages.

Das Ganze findet ihr bei Github und den fertigen Blog auf https://kuddl.github.io/

Ziele

Ich versuche hier im Blog meine Interessen rund um das Thema Webdevelopment zu sammeln und zu bündeln. Alles was dazu gehört. Von Console Tools bis zu Editor Erweiterungen, von .htaccess bis SVG-Sprites.

Erwartungen

  1. Der Blog soll statisch sein. Als nicht in PHP oder node.js.
  2. Resposive Web Design
  3. Gulp als Taskrunner
  4. Jade für HTML und Logik
  5. Markdown (CommonMark) zu schreiben
  6. Stylus für CSS
  7. Browserify für JS (evtl. mit npm, component)
  8. GIT als Versionierungssystem
  9. GIT als Workflow (Branches für "Artikel")
